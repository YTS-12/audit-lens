탭 분리 — 기존 화면 무변경

"분석 검색" 탭은 지금 화면 그대로입니다(스크리닝·조회, 단발 질문). 맥락 기억은 "AI 대화" 탭에만 존재해서, 단발 질의가 이전 맥락에 오염될 위험이 원천 차단됩니다. BETA 배지로 실험 기능임을 표시하고, 저장된 대화 목록은 "이전 대화 ▾" 드롭다운으로 엽니다.

기존 UX 보존
2

이전 답변 자동 접기

채팅 형식이지만 이전 문답은 "질문 → 결과 요약" 한 줄 카드로 자동으로 접히고, 최신 답변만 펼쳐진 상태를 유지합니다. 긴 보고서형 답변이 쌓여도 스크롤이 늘지 않고, 필요하면 "펼치기"로 그 자리에서 다시 봅니다.

긴 답변 해결
3

맥락 해석 표시

"그럼 작년은?"을 시스템이 어떻게 알아들었는지 답변 위에 한 줄로 보여줍니다. 잘못 알아들었으면 답을 기다릴 필요 없이 바로잡습니다.

신뢰 · 투명성
4

"기억 중" 칩 + 입력창

대화가 기억하는 회사·계정·조건을 입력창 위에 표시합니다. ×로 하나씩 잊게 하거나 "비우기"로 초기화. 입력창 문구가 사용법 안내를 겸해서 별도 튜토리얼이 필요 없습니다.

사용자 제어

구현 메모 — 그림이 코드와 어떻게 이어지나

  • 분석 검색 탭은 코드 변경이 없습니다. 지금의 /api/ask 흐름 그대로, 상태 없음(stateless). 대화 탭만 thread_id를 붙여 LangGraph 체크포인터(Postgres)에 상태를 저장합니다.
  • 이전 답변 접기는 프런트 처리입니다. 마지막 항목만 펼침 상태로 렌더링하고, "펼치기" 클릭 시 저장된 답변을 그 자리에서 다시 그립니다 — LLM 재호출 없음.
  • "기억 중" 칩은 상태(state)의 company · account · years 필드를 그대로 노출한 것입니다. × 클릭 = 해당 필드 삭제. 다른 회사를 물으면 자동 교체됩니다.
  • (선택) 두 탭 연결 — 분석 검색 답변 하단에 "이 결과로 대화 시작" 버튼 하나를 두면, 검색 결과를 컨텍스트로 물려받은 새 대화가 열립니다. 기존 화면에 추가되는 유일한 요소이며, 빼도 무방합니다.
